Reflection

There are several similarities and differences I have noticed between musicXML and the MEI files. First, MEI files are much more compact and concise. For example, measure 1 takes up 80 lines in musicXML and only 18 in MEI. This is because in musicXML, each component of the measure and notes, such as pitch, duration, octove, and stem direction, are separate elements on different lines (for example, "<octave>2</octave>"). In MEI, they are simply attributes on the same line as the note they are describing (oct="3"). This makes it much more concise and allows it to take up less storage space, but it can also limit comprehension since it requires deeper contextual analysis.

Second, musicXML files are much more descriptive in terms of format. For each note, it has "default-x" and "default-y" values, which fix the position of the note on the page, and the measure sizes are also set. In MEI, there is no such data that fixes the placement of notes. Consequently, changing the window size on the website changes the visual format of the notes, which can allow for more versatility but less control and difficult readability when the notes are too close together or too far apart.

Lastly, the notes are described differently in each format. In musicXML, note durations are encoded by time they take up. For example, a quarter note has duration 2 and element type is quarter. The same note in MEI would have a duration of 4 (quarter note) and no attribute signifying the type of note in words. A half note has duration 4 in musicXML and 2 in MEI, and an eighth note has duration 1 in musicXML and 8 in MEI. This means that they are inverse each other, so higher duration values in musicXML correspond to lower duration values in MEI. Each can be more intuitive than the other depending on the user, so you just need to make sure to understand what the values mean in each format.
